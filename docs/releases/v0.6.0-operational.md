## Architecture Overview

```
Browser / Next.js App Router
        │
        ▼
Server Actions  (app/(app)/**/actions.ts)
        │
        ▼
Services Layer  (lib/services/)
  ├── ProblemsService       orchestrates full DSA workflow
  ├── RevisionService       SM-2 scheduling + event emission
  ├── KnowledgeService      vault CRUD + entity linking
  ├── TaxonomyService       usage re-ranking + AI evolve
  ├── ActivityService       activity_logs + daily_logs
  ├── TimelineService       event → human-readable feed
  ├── AnalyticsService      read-only stats
  ├── AiService             battle plan (AI read model)
  ├── DashboardAggregationService  single read model for /overview
  ├── KnowledgeGraphService linked resources across entities
  ├── ConceptService        concept notes CRUD
  ├── RoadmapService        roadmap management
  ├── IitService            IIT course/lecture/assignment CRUD
  └── EventService          append-only domain event bus
        │
        ▼
Repository Layer  (lib/repositories/)
        │
        ▼
Supabase (PostgreSQL + RLS + Storage)
```

## Event Flow Diagram

```
Problem Solve (createLearningEntry)
  problem.created
    └─► attempt.create
         └─► problem.solved
              ├─► reflection.created         (if reflection supplied)
              ├─► code_version.created       (if code supplied)
              ├─► revision.scheduled         (always)
              ├─► roadmap.progressed         (if problem linked to roadmap items)
              └─► taxonomy.evolved           (background; reranks topics/patterns)

Revision Outcome (recordRevision)
  revision.succeeded  OR  revision.failed
  revision.snoozed    (snoozeRevision)

Concepts
  concept.created    (ConceptService.create)
  concept.revised    (ConceptService.update)

Knowledge Vault
  knowledge.created  (KnowledgeService.create)
  knowledge.updated  (KnowledgeService.update)
  knowledge.deleted  (KnowledgeService.remove)
  knowledge.linked   (KnowledgeService.link)

Taxonomy
  taxonomy.evolved      (TaxonomyService.evolve — after every solve)
  taxonomy.approved     (TaxonomyService.approve — user approves AI proposal)
  taxonomy.dismissed    (TaxonomyService.dismiss)
  taxonomy.auto_added   (TaxonomyService — high-confidence AI candidate)
  taxonomy.proposed     (TaxonomyService — low-confidence AI candidate)

Auth
  auth.login / auth.logout
```

## Known Limitations

1. **Migration 0013 not yet applied to production.** `knowledge_items` and `knowledge_links` are missing from the live DB. Apply `database/migrations/_production_bundle_phase5.sql` in the Supabase SQL Editor to resolve.
2. **Knowledge Vault file uploads deferred.** `storage_path` column exists; byte upload pipeline (concept-images bucket) not yet wired in Phase 5B.1.
3. **AI battle plan is deterministic, not LLM-generated.** `AiService.battlePlan()` derives the plan from data heuristics. A real LLM call is a Phase 5C deliverable.
4. **No background job runner.** Taxonomy evolve and AI re-scoring run synchronously on each solve. A cron-based trigger is recommended for production scale.
5. **Battle Plan module** — `ai_daily_briefs` table exists; the Battle Plan UI surfaced through `DashboardAggregationService` but no standalone `/battle-plan` page yet.
